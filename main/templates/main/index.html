Maple: A UI for Data Science

Welcome to the Maple demo site.

To log in and explore this site as a staff user, use 'testuser' and 'testpassword'.

Maple is a user interface focused web application to manage data science and analytics in organisations. Its sets out to solve the problem of tracking data assset, data elements users and user roles, and analytics work streams. It aim to increase the productivity for data scientists while ensuring transparency and security for modern data governance.

The demo version of this application application includes only fake user names and profiles, along with sample datasets taken from Kaggle. It and sample requests and projects. The data in maple is reset periodically.

The feature set of Maple includes:

Metadata repository
Track data assets, data users, and user roles. The backend also holds all data element information, and masterdata and reference data, and can be used for data lineage tracking.

Research and reporting streams
Track streams of work for research and reporting teams, and link this to data use and approval tracking.

Data request management
Manage ad-hoc data requests with task allocation and notes, and track real-time metrics through application Data APIs

Analytics server integration
Inegration to an R Shiny Server or R Connect server to host R Apps and Jupyter notebooks

API first design
Access all application data via REST APIs using Django Rest Framewoork or Swagger.

News and resources
Manage staff updates and alerts, and provide, and static information for staff accessing data, or undertaking research and analytics.

Architecture Overview
Maple is opinionated as a UI, but it aims to be highly flexible as an approach to analytics architecture. A high level overview of the architecture I usually use can be seen below and has the following components: a Django web application; an R Analytics Server that can serve R and Python notebooks, R function APIs, R Shiny applications; and a web API/microservices to manage the management of data and logging of data use.

The Maple project is well suited to scaleability. This version is being hosted Digital Ocean deployments, and is well suited to both single instance deployments with limited data assets, to cloud deployments using microservices such as Kubernetes.
