Research and Data Governance
Guidelines for undertaking research
CIDA holds a complete list of Research programs as well as thier associated ouputs being undertaken internally can be found here along with their related research outputs and links to ethics where applicable.

The steps needed research with a view to publishing you will need to do the following:
  1. Get approval for your director
  2. Onboard your proposed research using the template at EF/889/776
  3. Undertake a scoping and analysis
  4. Brief to concept brief approved
  5. Access the dataset


Note that the majority of research undertaken in the Institute utilises the Master Linked Dataset, a linked dataset which includes the NSW Cancer Registry, ACT Cancer Registry, NSW Notifiable Conditions Information Management System, ACT Communicable Notifiable Diseases Management System, Breast Screen NSW, NSW Pap Test Register, NSW Admitted Patient Collection, NSW Emergency Department Data Collection, NSW Registry of Births, Deaths and Marriages death registration, NSW Cause of Death Unit Record File, ACT Emergency Department Information System, ACT Registry of Births, Deaths and Marriages death registration, ACT Cause of Death Unit Record File, ACT Admitted Patient Care, the NSW Retrospective Radiotherapy Data Set, Medicare Benefits Schedule and Pharmaceutical Benefits Schedule

Note that if planning to use the MLD the Data Governance can onboard you into the Dataset, and it will not require Ethics amendments. This is a wide ranging protocol that we use for high volume research outputs.

Accessing data
Getting access
If you are an internal staff member seeking access any of the data assets for the purpose of analytics, you should request access through the Data Governance Team. In most cases, access is restricted to analytics data assets. For many data assets, there are extended usage notes which set out the conditions of internal and external use. You can also download a complete list of data assets and their associated metadata, or a complete list of users and roles for all data assets.

Coding resources
Getting started with R
The R programming language is open source, freely available, and highly to suited analytics tasks that range from descriptive statistics through to advanced machine learning. It also has wide ranging capabilities for creating interactive visualisations and camera-ready journal papers. R Studio (the IDE most commonly used to write R code) also has built in support for many languages, including SAS, Latex, C, C++, Haskell, Python, Javascript, HTML and SQL.

There are a number of excellent resources available to start learning R. DataCamp provides a free introductory course (found here) and there is also a getting started set of tutorials by Greg Hamel (found here. The Hamel tutorials are also available in Python using Jupyter notebooks, thus providing an opportunity to compare the differences between these two languages. Both Python and R are excellent choices for data science and it will not hurt to learn them both. If you are looking for good ways to ensure clean documentation of you code (which you should be doing!) also read through Hadley Wickham's article on auto-generating Rd files. The CIDA project will also be seeking to obtain a site wide licence for DataCamp which provides many data science courses in R and Python.

Resources can also be found on MOOC sites such as Coursera and EDx. An excellent starting point is the John Hopkins University Data Science Specialisation, 10 courses which cover a wide range of data science practics and feature R. There are also a number of other related Data Science courses from John Hopkins University and an excellent course in mathematics for machine learning from Imperial College London. All courses can be audited with no cost.

Another excellent resource is the Applications of Statistical Learning course by Trevor Hastie and Robert Tibshirani. This resource provides two in depth text books covering statistical and machine learning with applications in R, along with 15 hours of lectures. Note that a solid background in statistics is assumed, and concepts are explained through a statistical lens (many other Machine Learning courses tend to utilise ideas more directly from Calculus and Linear Algebra such as the one noted above). This is in invaluable resource however, and ideal to undertake after the John Hokpins courses.

Coding conventions
The Institute adheres to coding conventions for R as set out in the Google R Style Guide and adopts the Tidyverse. These style guides include naming and documentation conventions for variables and functions, extensive information for layout and ordering of code, and commonly used libraries and idioms. Note that if you are buidling Shiny Apps and want them to be hosted on the Institute Shiny Server, they must conform to this style guide. You will find this is not overly onerous and that it makes life easier for everyone, and will make your code cleaner and a joy to work with!

The most important aspect of writing code? Readability. This is crtical when writing code in any language and you should always code with this in mind. A great way to work on writing beautiful, clean code is to come along to the R Coders meetup group which meets weekly to pick up lots of great practices.

Using R for ETL
The CIDA project uses the R Package DPLYR for all data transformation tasks and does not use SQL. DPLYR acts as a data query API, and can connect directly to a wide range of different data formats (including SQL, MongoDB, SAS, CSV, Spark, Hadoop, Kafka etc) and is designed for optimised lazy loading and portability. It is also highly readable and testable. Data transformation tasks of any scope can be carried out using DPLYR and it can generate code in many other languages to carry out data manipulation tasks. An excellence tutorial to use DPLYR can be found on DataCamp.

Creating Shiny apps and Notebooks
One of the major strengths of R is its ability to create and deploy data analysis in a wide range of different formats. These include web applications, Notebooks, PDF, camera-ready journal papers, etc. If you are building these you should use the available templates which will be made available from this page sooon.

As part of the CIDA project, we are currently building a wide range of templates that can be used to quickly get up and running when building data transformation tasks, Shiny Apps, and Notebooks. These include templates for publish ready RBCO visualisations, standard approaches to building apps, notebooks and even ministerial briefs. Developing templates and accompanyting documentation is a great way to get explore the R language and if you want to get involved in this part of the project, contact Jamie Gabriel or Richard Walton.

R Coders meetup group
The R Coders meetup group happens every Friday at 11am on level 9 at the Cancer Institute NSW. Its the place to meet other developers, share your code, and explore what is happening in data science. It has broad membership, from those starting out from other languages and frameworks, and moving into data science, through to seasoned analysts working in R. Its a supportive and friendly environment and while there is an emphasis on the R language, we also explore Python, SAS and a number of other languages.
